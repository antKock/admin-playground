1. State Machine — All Rule Field States

The rule field has exactly 4 states. No "collapse" mechanic — the field is either in read mode or edit mode.

4 states total:
① Texte Read — non-empty rule, read-only prose with syntax colors, click anywhere to enter edit mode
② Texte Edit — CodeMirror active, cursor blinking, same syntax colors, autocomplete available
③ JSON Read — prose mirror shown (read-only), click to edit JSON below

On page load: check if rule field value is empty/null. If empty → state ②. If non-empty → state ①. No localStorage persistence needed — purely based on content.

When a rule is empty, the field immediately opens in Texte edit mode with a placeholder and cursor. No action needed from the admin.

4. Edit Mode — Texte Editing

In edit mode, the admin types constrained French prose with real-time syntax coloring and autocomplete. Colors are identical to read mode.

Simple rule with dot-path variable EDITING
Règle de visibilité
formation.type_etablissement = 'université'
et candidat.age ≥ 18
et statut fait partie de ['actif', 'en attente']
Valide
Rule with nested parenthetical OR EDITING
Règle de visibilité
age ≥ 18 et age ≤ 65
et (departement fait partie de ['75', '92', '93']
    ou region = 'IDF')
Valide
Dot-path variables: Variables like formation.type_etablissement or candidat.age are colored as a single token. Autocomplete suggests the full path.
Parenthetical grouping: (...) creates nested OR/AND. Matching parens highlight on cursor proximity (CodeMirror bracket matching).
Line breaks: Enter key for readability — line breaks are whitespace, don't affect parsing.

5. Complex Rule — Read & Edit (OR Bullets, Quantifiers, Nested Groups)

The full complexity example from the exploration doc. Shows how top-level OR renders as bullets in read mode, and how quantifiers (some/all/none) appear in both modes.

Key pattern from jsonlogic-prose.ts: Top-level OR branches render as bullet points (•). Each bullet is an independent AND-chain. Nested OR within a bullet uses inline parentheses (x ou y). This applies to both read mode and edit mode.
Complex rule — Read Mode DEFAULT ON PAGE LOAD
Règle de visibilité
Le paramètre est activé si au moins une de ces conditions est vraie :
age ≥ 18 et age ≤ 65 et (departement fait partie de ['75', '92', '93', '94'] ou region = 'IDF') et nombre_unite × cout_unitaire > 1000
au moins un élément de beneficiaries satisfait : departement = '75' et subvention_locale + subvention_region ≥ 500 et aucun champ manquant parmi ['numero_siret']
presta = 'OK' et (aide_sollicitee < aide_sollicitable ou (statut = 'prioritaire' et budget_total > 10000)) et type_projet ne contient pas 'refuse'
↓ Click anywhere — enters edit mode ↓
Complex rule — Edit Mode (same colors, same structure) EDITING
Règle de visibilité
age ≥ 18 et age ≤ 65 et (departement fait partie de ['75', '92', '93', '94'] ou region = 'IDF') et nombre_unite × cout_unitaire > 1000

au moins un élément de beneficiaries satisfait : departement = '75' et subvention_locale + subvention_region ≥ 500 et aucun champ manquant parmi ['numero_siret']

Valide — 3 branches OR
Read mode vs Edit mode for OR:
• Read mode: Top-level OR branches are rendered with bullet points (•) and indented — mirroring the exact output of jsonlogic-prose.ts. The prefix says "au moins une de ces conditions est vraie".
• Edit mode: No bullets — the admin types plain text. OR branches are separated by blank lines for readability. The parser identifies top-level OR branches by the structure (lines that are independent AND-chains separated by blank lines).

Quantifiers in edit mode: Phrases like au moins un élément de, tous les éléments de, aucun élément de are typed as plain text. They are colored as keywords (tk-kw). The parser recognizes them as quantifier operators.

Nested parens: (aide_sollicitee < aide_sollicitable ou (statut = 'prioritaire' et budget_total > 10000)) — double nesting is fully supported. CodeMirror bracket matching highlights paired parens.
The parser needs to handle top-level separation: blank lines between OR branches in edit mode. In read mode, use the bullet-branch CSS class for each top-level OR branch. The prefix "Le paramètre est activé si au moins une de ces conditions est vraie :" is only shown in read mode when there are multiple OR branches. For a single branch (no OR), the prefix is just "Le paramètre est activé si :".

Autocomplete trigger rules:
• Start of line / after connector → suggest variable names (fuzzy) + quantifier expressions
• After a variable name → suggest operators filtered by variable type ("Opérateurs pour nombre", "Opérateurs pour texte")
• After an operator → no dropdown (admin types value freely). Exception: after fait partie de → suggest [
• After a complete condition → suggest connectors (et, ou)
• After "au moins un élément de" → suggest only array-type variables from dictionary
• After quantifier + array variable + "satisfait :" → suggest variable names again (new sub-condition)

Quantifiers in autocomplete: When the context expects a variable or expression start, the dropdown shows both individual indicators AND quantifier phrases ("au moins un élément de …", "tous les éléments de …", "aucun élément de …") in a separate "Expressions" group. Selecting a quantifier phrase inserts the text and then triggers a second autocomplete for the array variable.

Keyboard: ↑/↓ navigate. Enter/Tab accept. Esc dismiss. First item pre-selected.
Fuzzy matching: contien → contient. forma → formation.type_etablissement.
Use @codemirror/autocomplete. Source function receives editor state + cursor position. Parse tokens left of cursor to determine context. Header text changes per context. For quantifiers: after inserting "au moins un élément de ", immediately trigger a new autocomplete showing only array-type indicators. After array variable + " satisfait : ", reset context to expect a new condition.

Validation uses two complementary signals: a compact badge for overall status, and inline wavy underlines with hover tooltips for specific errors. Only one tooltip visible at a time (the one being hovered).

Two-layer error reporting:
Layer 1 — Inline underlines: Red wavy underline on each offending token. All errors are underlined simultaneously.
Layer 2 — Hover tooltip: Hovering a specific underlined token shows ONE dark tooltip with an actionable message. Only one tooltip visible at a time — the one currently being hovered. No tooltip shown by default.
Layer 3 — Badge: Right-aligned badge with count: 2 erreurs — survoler pour détails. Tells the admin to hover for specifics.

Error types and tooltip messages:
• Unknown variable → "Variable inconnue « X » — vérifiez l'orthographe"
• Missing value → "Valeur manquante après l'opérateur « > »"
• Type mismatch → "L'opérateur « > » n'est pas applicable au type texte"
• Unclosed parens → "Parenthèse non fermée"
• Dangling connector → "Condition manquante après « et »"
Use @codemirror/lint. Each diagnostic: {from, to, severity, message}. Tooltip = CodeMirror's built-in lint tooltip (shows on hover, one at a time). Badge is a custom panel below the editor, updated on every lint cycle.

10. Color Legend & Syntax Tokens

Every token type has a dedicated color. No bold is used. Keywords and operators share the same color. Identical colors in read mode and edit mode.

Token Type Color Examples CSS class
Variable #7c3aed statut   montant   formation.type_etablissement   candidat.age .tk-var
Keyword / Operator #555555 fait partie de   et   ou   ne contient pas   contient   >   ≤   =   ≠   ×   +   au moins un élément de   satisfait :   aucun champ manquant parmi .tk-kw
Value (string, number, array) #059669 'actif'   1000   ['75', '92', '93']   'IDF' .tk-val
Prefix (read mode only) #888888 italic Le paramètre est activé si : .tk-pfx
Error token #b32020 + underline foobar .tk-err + .error-underline
Bullet (read mode only) #555555 • (top-level OR branch marker) .bullet-branch::before
Key design decision: Keywords (fait partie de, et, ou, au moins un élément de) and symbol operators (>, ≤, =, ×) share the same #555555 color. This reduces visual noise — the important contrast is between variables (purple), values (green), and everything else (gray).
