## Successes & Strengths

1. **100% completion with zero regressions** — Full migration of 2 entities to a new architecture with all existing functionality preserved
2. **Progressive story sequencing** — Scaffold (0.1) → Shared utility (0.2) → Pilot (0.3) → Replicate (0.4) → Cleanup (0.5) was a velocity multiplier. Story 0.4 went significantly faster than 0.3 because the pattern was proven
3. **`withCursorPagination<T>()` as reusable foundation** — Built once in Story 0.2, composed into every domain store. 15 unit tests ensure reliability
4. **Code reviews caught meaningful improvements** — Per-mutation status signals, `effect()`-based form loading, StatusBadge in list views were all added during review
5. **Clean ACTEE separation held up** — No component imports a store directly. Facade is the single UI entry point. Feature stores are read-only computed. Architecture validated
6. **Solid test strategy** — Shared utility tests + per-entity domain store + facade tests. Zero regressions across the epic

---

## Challenges & Growth Areas

1. **NgRx Signals v21 generic typing friction** — `signalStoreFeature` with generics requires `as never` casts for `patchState` due to TypeScript index signature constraints. Appeared in 3/5 stories (0.2, 0.3, 0.4)
2. **inject() context gotcha** — `inject(HttpClient)` inside `withCursorPagination` loader fails when running inside rxMethod's switchMap (outside Angular injection context). Required `withProps(() => ({ _http: inject(HttpClient) }))` workaround. Discovered in Story 0.3
3. **Vitest/zone.js incompatibility** — `fakeAsync`/`tick` don't work in Vitest environment. Tests restructured to use synchronous `of()` observables. Discovered in Story 0.2
4. **Story ACs didn't fully capture ACTEE requirements** — Per-mutation status signals, effect()-based form loading, and StatusBadge integration were caught in code review, not in original story specs

---

## Key Insights & Learnings

1. **Pilot-then-replicate is a proven acceleration pattern** — Story 0.3 (pilot) was the heaviest; Story 0.4 (replication) was significantly faster. Apply this to all future entity work
2. **Code review is a critical safety net, not a formality** — It caught ACTEE pattern gaps that story specs missed. Keep mandatory
3. **NgRx Signals advanced composition has rough DX edges** — Custom generic store features are powerful but require workarounds. Document upfront
4. **Framework workarounds should be documented immediately** — Rediscovering the same issue across multiple stories wastes effort
5. **Well-calibrated story scope matters** — Each story had a focused scope. No story tried to do too much. The 5-story structure was right-sized
